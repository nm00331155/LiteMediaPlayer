LiteMediaPlayer Explorer UI/UX 改善タスク
あなたはこのリポジトリの既存コードを理解した上で、Explorer機能のUI/UXを全面的に改善する実装アシスタントです。
目的は、現在すでに存在している Explorer 機能をベースに、シンプルだがグラフィカルで、誰でも直感的に使えるファイル/メディアエクスプローラーUIへ進化させることです。

最優先のゴール
現在の Explorer は機能自体はあるが、見た目が簡素で使いづらい
UI は ミニマル だが 安っぽく見えない
UX は 迷わない / すぐ使える / 次に何をすればいいか明確
既存機能を壊さず、段階的に改善
LiteMediaPlayer 全体の体験として、「閲覧 → 選択 → 再生」までの動線が最短になるようにする
作業方針
まず最初に以下を実施してください。

リポジトリ全体を確認し、Explorer に関連する既存ファイルを特定する
UI定義、状態管理、イベント処理、ファイル読み込み、再生連携箇所を洗い出す
既存の命名規則、コンポーネント構造、スタイル実装方法に合わせる
いきなり全面破壊せず、既存機能を維持したまま差し替え可能な形で改善
実装前に、どのファイルをどう変更するかを簡潔に整理してから着手する
改善対象のExplorerの理想像
Explorerは単なるファイル一覧ではなく、メディア選択のための操作ハブとして設計すること。

以下の体験を重視すること：

ユーザーが開いた瞬間に何ができるか分かる
フォルダ移動、戻る、再読込、絞り込みが自然にできる
ファイルとフォルダの区別が一目で分かる
メディアファイルが視覚的に見つけやすい
クリック時の挙動が予測しやすい
再生までのアクション数が少ない
選択状態・ホバー状態・現在地が明確
空状態や読み込み中も不親切にならない
必須UI要件
Explorer UI を以下の構成で改善してください。

1. ヘッダー / ツールバー
最低限以下を含めること：

ダブルクリックまたは明確な操作でフォルダ移動 / 再生ができる
キーボード操作にも配慮（上下、Enter、Backspace など可能な範囲）

必須UX要件
1. 動線設計
Explorer を開いてから再生までの最短動線を最重要視すること。

理想フロー：

Explorer を開く
目的のフォルダを見つける
対象ファイルを選ぶ
ワンクリックまたはダブルクリックで再生
この流れを阻害する不要な確認ダイアログや過剰演出は避けること。

2. 視認性
コントラスト不足を避ける
選択状態 / hover / focus を明確化
情報量は増やしてもノイズにしない
余白、区切り線、階層感で整理する
3. 学習コスト削減
初見で意味が分かる UI を優先
独自すぎる操作は避ける
OSの一般的なファイルエクスプローラーに近い期待に沿うこと
ただし見た目は現代的に整えること
4. 気持ちよさ
クリック反応は軽快に
hover / selection / transition は短く控えめ
過度なアニメーションは不要
「ちゃんと反応した」と感じるフィードバックは入れる

追加で強く意識してほしいこと
このExplorerは、単なる一覧表示ではなく、LiteMediaPlayerの入口体験です。
そのため「見た目を整える」だけでなく、以下を重視してください。

開いた瞬間に使い方が分かる
目的ファイルへ最短で辿り着ける
選ぶ・開く・再生するが自然につながる
シンプルなのに安っぽくない
毎日使っても疲れない
既存コードを読んだ上で、最小の破壊で最大の体験改善を行ってください。

実装品質ルール
UIだけ差し替えて終わりにしないこと
使いやすさに直結する状態設計まで見直すこと
コンポーネントの責務を整理すること
再利用しやすい小さなUI部品に分けること
ただし過剰分割は避けること
「見た目は良いが使いにくい」状態を作らないこと
実装後にセルフレビューし、UX上の弱点を列挙して必要なら追加修正すること
セルフレビュー項目
実装後、自分で以下を確認してください。

初見ユーザーが迷う箇所はないか
どのボタンが何をするか見て分かるか

フォルダ遷移がストレスなくできるか
検索導線が自然か
空状態が不親切でないか
一覧の密度が高すぎないか
見た目だけでなく再生動線が改善されているか
実装が将来の機能追加を邪魔しないか
問題があれば修正してから完了すること。
