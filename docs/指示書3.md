言語フラッシュループ — 根本原因と完全修正
何が起きているか
前回の修正で MainActivity から LaunchedEffect は削除されましたが、ループは止まっていません。理由は以下の連鎖です：

1. SettingsScreen で「English」をタップ
2. SettingsViewModel.updateLanguage() が呼ばれる
3. appSettingsStore.updateLanguage(EN) → DataStore に "EN" 保存
4. LocaleHelper.applyLanguage() → AppCompatDelegate.setApplicationLocales("en")
5. AppCompatDelegate が Activity を自動再生成 (recreate)
6. Activity 再生成 → Compose 再構築 → SettingsScreen 再表示
7. SettingsScreen の SettingsViewModel が再作成される
8. uiState が settingsFlow から再取得 → language = EN
9. SettingsScreen の OptionChips で EN が selected として表示される
   → ★ここまでは正常

問題: Activity 再生成時に AppCompatDelegate のロケールと
DataStore のロケールが一瞬不整合を起こし、
settingsFlow の初期値 AppSettingsState() (language=SYSTEM) が一瞬流れる
→ OptionChips が SYSTEM を選択状態で描画
→ しかし DataStore の読み込みが完了すると language=EN に変わる
→ recompose 発生 → 画面が一瞬切り替わる（フラッシュ）

さらに AppCompatDelegate.setApplicationLocales() は
「現在のロケールと異なるロケールが設定された場合」に Activity を再生成する。
Activity 再生成後、AppCompatDelegate の内部状態が "en" なのに
DataStore がまだ読み込み完了していない初期状態 (SYSTEM) が流れ、
もし SYSTEM がデバイスのロケール "ja" に解決されると
「現在 en なのに ja を設定しようとしている」と判断してまた再生成…
→ 無限ループ
修正方針
AppCompatDelegate.setApplicationLocales() を使わない。 代わりに Per-App Language API（Android 13+）のみを使い、Android 12以下では Configuration を直接変更する方式に切り替える。ただし最もシンプルで確実な方法は、言語切替を AppCompatDelegate に完全に任せ、DataStore には保存しないこと。

しかしこのアプリでは DataStore で統一管理しているため、DataStore の言語設定は保存のみ行い、実際のロケール適用は Activity 起動時に1回だけ行う方式にします。

package com.example.litemediaplayer.settings

import android.app.LocaleManager
import android.content.Context
import android.os.Build
import android.os.LocaleList
import androidx.appcompat.app.AppCompatDelegate
import androidx.core.os.LocaleListCompat

object LocaleHelper {

    /**
     * 現在適用されているロケールを取得（AppCompatDelegate から）
     */
    fun getCurrentAppliedLanguage(): AppLanguage {
        val locales = AppCompatDelegate.getApplicationLocales()
        if (locales.isEmpty) return AppLanguage.SYSTEM
        return when (locales[0]?.language) {
            "ja" -> AppLanguage.JA
            "en" -> AppLanguage.EN
            else -> AppLanguage.SYSTEM
        }
    }

    /**
     * 言語を適用する。
     * ただし「現在既に同じ言語が適用されている場合は何もしない」ことで
     * Activity再生成ループを防止する。
     * 
     * @return true=ロケールが変更されActivity再生成が発生する, false=変更なし
     */
    fun applyLanguageIfNeeded(context: Context, language: AppLanguage): Boolean {
        val current = getCurrentAppliedLanguage()
        if (current == language) {
            // 既に同じロケールが適用済み → 何もしない
            return false
        }
        applyLanguageForce(context, language)
        return true
    }

    /**
     * 強制的にロケールを適用する（Activity再生成を引き起こす可能性あり）
     * 呼び出し元が重複チェック済みの場合のみ使用すること
     */
    private fun applyLanguageForce(context: Context, language: AppLanguage) {
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
            try {
                val localeManager = context.getSystemService(LocaleManager::class.java)
                if (language == AppLanguage.SYSTEM) {
                    localeManager.applicationLocales = LocaleList.getEmptyLocaleList()
                } else {
                    localeManager.applicationLocales =
                        LocaleList.forLanguageTags(language.toLanguageTag())
                }
                return
            } catch (_: Exception) {
                // フォールバック
            }
        }

        // Android 12以下、または Android 13 で例外が出た場合
        val tag = if (language == AppLanguage.SYSTEM) "" else language.toLanguageTag()
        AppCompatDelegate.setApplicationLocales(
            LocaleListCompat.forLanguageTags(tag)
        )
    }

    private fun AppLanguage.toLanguageTag(): String {
        return when (this) {
            AppLanguage.JA -> "ja"
            AppLanguage.EN -> "en"
            AppLanguage.SYSTEM -> ""
        }
    }
}

import android.os.Bundle
import androidx.activity.compose.setContent
import androidx.activity.enableEdgeToEdge

import androidx.compose.material3.Surface

import com.example.litemediaplayer.navigation.AppNavigation
import com.example.litemediaplayer.settings.AppSettingsStore
import com.example.litemediaplayer.settings.AppSettingsState
import com.example.litemediaplayer.settings.LocaleHelper
import com.example.litemediaplayer.settings.OrientationController
import com.example.litemediaplayer.settings.ThemeMode
import com.example.litemediaplayer.ui.theme.LiteMediaTheme
import dagger.hilt.android.AndroidEntryPoint
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.compose.collectAsStateWithLifecycle
import kotlinx.coroutines.flow.first
import kotlinx.coroutines.runBlocking
import javax.inject.Inject

@AndroidEntryPoint
class MainActivity : AppCompatActivity() {
    @Inject
    lateinit var appSettingsStore: AppSettingsStore

    override fun onCreate(savedInstanceState: Bundle?) {
        // ★ setContent の前に、DataStore から言語設定を同期的に読み取り、
        //   現在のロケールと異なる場合のみ適用する。
        //   これは Activity の onCreate で 1 回だけ実行される。
        //   applyLanguageIfNeeded は「既に同じなら何もしない」ので
        //   再生成ループは発生しない。
        if (savedInstanceState == null) {
            // 初回起動時のみ（再生成時は savedInstanceState != null なのでスキップ）
            try {
                val settings = runBlocking { appSettingsStore.settingsFlow.first() }
                LocaleHelper.applyLanguageIfNeeded(this, settings.language)
            } catch (_: Exception) {
                // DataStore 読み込み失敗時は何もしない
            }
        }

        super.onCreate(savedInstanceState)

        OrientationController(
            activity = this,
            appSettingsStore = appSettingsStore
        ).observe()

        enableEdgeToEdge()
        setContent {
            MainRoot(appSettingsStore = appSettingsStore)
        }
    }
}

@Composable
private fun MainRoot(appSettingsStore: AppSettingsStore) {
    val appSettings by appSettingsStore.settingsFlow.collectAsStateWithLifecycle(
        initialValue = AppSettingsState()
    )

    val darkTheme = when (appSettings.themeMode) {
        ThemeMode.LIGHT -> false
        ThemeMode.DARK -> true
        ThemeMode.SYSTEM -> isSystemInDarkTheme()
    }

    // ★ ここでは言語適用を一切行わない。
    //   テーマのみ Compose で反映する。

    LiteMediaTheme(darkTheme = darkTheme) {
        Surface(modifier = Modifier.fillMaxSize()) {
            AppNavigation()
        }
    }
}

なぜ前回の修正では直らなかったか
前回は MainActivity から LaunchedEffect を削除しましたが、SettingsViewModel.updateLanguage() 内の LocaleHelper.applyLanguage() が無条件で AppCompatDelegate.setApplicationLocales() を呼ぶ点が残っていました。Activity 再生成後に DataStore の settingsFlow が値を emit し、それが Compose の recomposition を起こすだけなら問題ないのですが、settingsFlow の initialValue = AppSettingsState() が language = SYSTEM として一瞬 emit されることが問題でした。

今回の修正のポイントは3つです：

ポイント1: applyLanguageIfNeeded() で「現在のロケールと同じなら何もしない」ガードを入れた。これにより、Activity 再生成後に同じ言語を再適用してさらに再生成…というループが不可能になった。

ポイント2: MainActivity.onCreate() で savedInstanceState == null（初回起動時のみ）の場合だけ言語を適用する。Activity 再生成時（savedInstanceState != null）はスキップするので、再生成の連鎖が起きない。

ポイント3: SettingsViewModel.updateLanguage() で、ユーザーが実際にタップした言語が現在と同じなら即 return する。

この3重のガードにより、どのパスからも無限ループは発生しません。
