# LiteMediaPlayer v14 修正指示書

> **重要**: `docs/` 配下の MD ではなく `.kt` ソースファイルを直接編集すること。

---

## 問題一覧と根本原因

| # | 問題 | 根本原因 |
|---|------|---------|
| A | プレイヤー：フォルダロックをオンにしても非表示にならない | `buildFolderShells` は `isHidden && isLocked` のフォルダを除外する仕組みだが、`FolderManagerScreen` で `toggleLock` しても `isHidden = false` のまま。`isHidden` が `true` にならない限り非表示にならない。5回タップは `hiddenLockContentVisible` を切り替えるだけで、フォルダ一覧のフィルタリング自体が正しく機能していない |
| B | プレイヤー：シーク映像プレビューが表示されない | `seekRetrieverState` の初期化で `MediaMetadataRetriever` が一部のcontent URIでフレーム取得に失敗する。`OPTION_CLOSEST_SYNC` だけでは取得できない端末/動画がある。また `seekRetriever` が `null` の場合にフォールバックがない |
| C | コミック：フォルダロックの鍵マークを押しても意味がない | `toggleComicFolderLock` は DB に `LockConfig` を書き込むが、`ComicShelfScreen` のフォルダ一覧表示時にロック状態をチェックしていないため、ロックされたフォルダも普通に表示・アクセスできる。5回タップも同様に `hiddenLockContentVisible` を切り替えるだけで、フォルダのフィルタリングに反映されていない |
| D | コミック：本を開く際のローディング表示がない | `openBook` は IO スレッドでページ読み込みを行うが、UI に読み込み中状態が反映されない |

### 7-2: フレーム取得で複数の方法を試す

**現在のプレビュージョブ**:
```kotlin

    delay(50)

        runCatching {
            seekRetriever?.getFrameAtTime(
```

**変更後**:
```kotlin
previewJob = previewScope.launch {
    delay(80) // デバウンス
    val positionMs = value.toLong()
    val positionUs = positionMs * 1_000L
    val bitmap = withContext(Dispatchers.IO) {
        val retriever = seekRetriever ?: return@withContext null

        // 方法1: OPTION_CLOSEST_SYNC (高速、最寄りのキーフレーム)
        var frame = runCatching {
            retriever.getFrameAtTime(positionUs, MediaMetadataRetriever.OPTION_CLOSEST_SYNC)
        }.getOrNull()

        // 方法2: OPTION_CLOSEST (正確だが遅い)
        if (frame == null) {
            frame = runCatching {
                retriever.getFrameAtTime(positionUs, MediaMetadataRetriever.OPTION_CLOSEST)
            }.getOrNull()
        }

        // 方法3: OPTION_PREVIOUS_SYNC
        if (frame == null) {
            frame = runCatching {
                retriever.getFrameAtTime(positionUs, MediaMetadataRetriever.OPTION_PREVIOUS_SYNC)
            }.getOrNull()
        }

        // 方法4: デフォルト (位置 0 のフレーム)
        if (frame == null && positionMs > 0) {
            frame = runCatching {
                retriever.getFrameAtTime(0L, MediaMetadataRetriever.OPTION_CLOSEST_SYNC)
            }.getOrNull()
        }

        if (frame == null) {
            AppLogger.w("SeekPreview", "All frame extraction methods failed at ${positionMs}ms")
        }

        frame
    }
    seekPreviewBitmap = bitmap
}
```

## 補足: グローバルロック設定が未設定の場合

`toggleComicFolderLock` と `FolderManagerViewModel.toggleLock` は、`APP_GLOBAL` の `LockConfig` が `null` の場合にエラーメッセージを表示して終了します。ロック機能を使うには、先に設定タブで何らかのロック設定（PIN等）を行う必要があります。

**もしグローバル設定なしでもロックを有効にしたい場合**は、`global == null` 時にデフォルトの `LockConfig` を作成するように変更してください:

```kotlin
if (global == null) {
    // デフォルトのロック設定で作成
    lockConfigDao.upsert(
        LockConfig(
            targetType = LockTargetType.COMIC_FOLDER.name,
            targetId = folderId,
            authMethod = "BIOMETRIC",
            pinHash = null,
            patternHash = null,
            autoLockMinutes = 5,
            isHidden = true,
            isEnabled = true
        )
    )
    return@launch
}
```

これは `FolderManagerViewModel.toggleLock` 側にも同様に適用してください。
