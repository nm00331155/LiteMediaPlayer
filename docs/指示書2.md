問題1: 動画再生ができない
根本原因: PlayerPlaybackScreen に渡される videoUri が 二重デコードされている。

Navigation.kt で Uri.encode(videoUri) してナビゲーションし、composable 内で Uri.decode(encodedUri) しているが、PlayerPlaybackScreen 内でも再度 Uri.decode(videoUri) している。SAF の content:// URI はこの二重デコードで壊れる。さらに MediaItem.fromUri() に渡す前の toUri() 変換も不安定。

## 問題2: サムネイルが表示されない

**根本原因**: `CoilConfig` に `SafVideoThumbnailFetcher` と `VideoFrameDecoder` の両方が登録されているが、**`SafVideoThumbnailFetcher` の `Factory` が `Uri` 型を受け取るのに対し、`AsyncImage` の `model` には `ImageRequest` を渡している**。Coil の Fetcher.Factory は `data` の型でマッチングするが、`ImageRequest` 内の `data` は `video.uri`（`Uri` 型）なので `SafVideoThumbnailFetcher` がマッチする。しかし、`SafVideoThumbnailFetcher` が `ContentResolver.loadThumbnail()` で失敗した場合、`VideoFrameDecoder` にフォールバックされず例外で終了する。

さらに `loadThumbnail` は **動画ファイルの MIME type が ContentResolver に認識されない**場合に失敗する。SAF 経由の TreeUri → 子ファイル URI では、MIME type が正しく返らないケースがある。

### Copilotへの指示:

サムネイル表示を修正してください

全体を以下に置き換え:

Copypackage com.example.litemediaplayer.core.memory

import android.content.Context
import android.graphics.Bitmap
import android.graphics.drawable.BitmapDrawable
import android.media.MediaMetadataRetriever
import android.net.Uri
import android.os.Build
import android.util.Size
import coil.ImageLoader
import coil.decode.DataSource
import coil.fetch.DrawableResult
import coil.fetch.FetchResult
import coil.fetch.Fetcher
import coil.request.Options
import java.io.IOException
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext

class SafVideoThumbnailFetcher(
    private val context: Context,
    private val uri: Uri
) : Fetcher {

    override suspend fun fetch(): FetchResult = withContext(Dispatchers.IO) {
        val bitmap = tryLoadThumbnail()
            ?: throw IOException("Failed to load thumbnail for $uri")

        DrawableResult(
            drawable = BitmapDrawable(context.resources, bitmap),
            isSampled = true,
            dataSource = DataSource.DISK
        )
    }

    private fun tryLoadThumbnail(): Bitmap? {
        // 方法1: Android 10+ の ContentResolver.loadThumbnail
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q) {
            try {
                return context.contentResolver.loadThumbnail(
                    uri, Size(320, 180), null
                )
            } catch (_: Exception) {
                // フォールバック
            }
        }

        // 方法2: MediaMetadataRetriever (FileDescriptor経由)
        try {
            context.contentResolver.openFileDescriptor(uri, "r")?.use { pfd ->
                val retriever = MediaMetadataRetriever()
                try {
                    retriever.setDataSource(pfd.fileDescriptor)
                    return retriever.getFrameAtTime(
                        1_000_000,
                        MediaMetadataRetriever.OPTION_CLOSEST_SYNC
                    )
                } finally {
                    runCatching { retriever.release() }
                }
            }
        } catch (_: Exception) {
            // フォールバック
        }

        // 方法3: MediaMetadataRetriever (URI直接)
        try {
            val retriever = MediaMetadataRetriever()
            try {
                retriever.setDataSource(context, uri)
                return retriever.getFrameAtTime(
                    1_000_000,
                    MediaMetadataRetriever.OPTION_CLOSEST_SYNC
                )
            } finally {
                runCatching { retriever.release() }
            }
        } catch (_: Exception) {
            // 全方法失敗
        }

        return null
    }

    class Factory(private val context: Context) : Fetcher.Factory<Uri> {
        override fun create(
            data: Uri,
            options: Options,
            imageLoader: ImageLoader
        ): Fetcher? {
            // content:// スキームの場合のみこの Fetcher を使う
            if (data.scheme == "content") {
                return SafVideoThumbnailFetcher(context, data)
            }
            return null
        }
    }
}

## 問題4: 言語切替でフラッシュループ

**根本原因**: `MainActivity.kt` 内の `LaunchedEffect(appSettings.language)` が **言語変更のたびに `LocaleHelper.applyLanguage()` を呼び、`AppCompatDelegate.setApplicationLocales()` が Activity を再生成し、再生成された Activity が再び `LaunchedEffect` を実行し…**という無限ループ。

特に `AppCompatDelegate.setApplicationLocales()` は Activity の recreation をトリガーする。再生成後に DataStore から言語設定を読み込み → 再び `applyLanguage` → また再生成…というループが発生している。

Copyfun updateServer(serverId: Long, input: NetworkServerInput) {
    viewModelScope.launch(Dispatchers.IO) {
        if (input.name.isBlank() || input.host.isBlank()) {
            postStatus("名前とホストは必須です")
            return@launch
        }
        // 既存のIDを持つエンティティとして保存 → upsert でUPDATE
        val entity = input.toEntity().copy(id = serverId)
        networkServerDao.upsert(entity)
        postStatus("接続先を更新しました")
    }
}
